Git ir Github

Git - kodo versijavimo programa.
Github - versijuoto kodo platforma.

Git komandos

  • git innit - sukurti git repozitorija
  • git status - pateikia git repozitorijos dabartine informacija
  • git add - prideda nurodytus failus prie commit paruostiems failams
  • git rm --cached - || git restore --staged - isima nurodytus failus is commitinimui paruostu failu
  • git commit -m "zinute apie commita" - sukuria kodo versija is staged failu su komentaru
  • git restore - sugrazina nurodyta faila i pradine busena (jei tie failai nebuvo staginti)
  • git push - nusiuncia lokalios repozitorijos commitus i isorine repozitorija
  • git pull - atsiuncia isorines repozitorijos pakitimus i lokalia repozitorija
  • git log - isveda commitu sarasa
  • git clone - nuklonuoja nurodyta projekta i dabartini aplankala
  • git checkout commitID|branchName - pereina i nurodyta kodo versija arba kita saka
  • git branch naujosSakosPavadinimas - sukuriama nauja saka
  • ...

Branch protection rules

Vienas svarbiausiu saku apsaugos taisykliu yra pull request (bandymas vienos sakos commitus sukelti i kita saka (merge)).
Pull Request taisykles viduje dar reiketu suteikti require approvals nustatyma, kuris reikalauja, kad nurodytas kiekis kitu projekto dalyviu patvirtintu ta pull requesta, norint, kad sakos butu apjungtos.

Eiliskumas pull request

Apsiemiau daryti kazkokia uzduoti. Kokie mano tolimesni veiksmai?

  1. Susikurti nauja saka - git branch naujaSaka
  2. Nueiti i susikurta saka - git checkout naujaSaka
  3. Atlieki uzduoti kuria apsiemei daryti. Su vienu ar keliais commitais.
    1. Add'ini pakitimus - git add .
    2. Comittini pakitimus - git commit -m "Pakeitimu info"
  4. Nueiti i main/master saka - git checkout main/master
  5. Atnaujinti main/master saka - git pull
  6. Grizti i savo saka git - checkout naujaSaka
  7. Sujungti savo saka su main/master saka ir isspresti galimai atsiradusius konfliktus - git merge main
  8. Nusiunti savo darbines sakos pakitimus i githuba - git push
  9. Sukurti pull requesta githube
  10. Kiti komandos nariai patvirtina/atmeta Pull requesta
  11. Jeigu pull requestas sulauke pakankamai patvirtinimu - vykdyti merge
